# wow-podcast-assets

## 用途

为 wow.insight 频道 (mio 主播) 的论文拆解 / 评测文案提供**永久可访问的图片 + 静态资源 CDN**。

- **图床后端**: GitHub `seleman66eeddwegger3-art/wow-podcast-assets` (public)
- **CDN 入口**: jsDelivr — `https://cdn.jsdelivr.net/gh/seleman66eeddwegger3-art/wow-podcast-assets@main/<path>`
- **本地路径**: 仍在 `~/PAPER-VAULT/<slug>/figures/` + `~/PAPER-VAULT/<slug>/动态效果.html`

## 推送流程 (SOP)

1. **新论文产出**: 在 `~/PAPER-VAULT/<slug>/` 跑 paper-podcast-prep pipeline
2. **Stage 5 配图下载**: `curl -L -o figures/<img>.png https://github.com/OpenBMB/MiniCPM/raw/main/assets/<img>.png` (或类似官方 URL)
3. **本地编译**: 校对 .md + figures 配对
4. **改 CDN URL**: `figures/xxx.png` → `https://cdn.jsdelivr.net/gh/seleman66eeddwegger3-art/wow-podcast-assets@main/<slug>/xxx.png`
5. **推送到本仓库**:
   ```bash
   cd /tmp/wow-assets/wow-podcast-assets
   mkdir -p <slug>
   cp ~/PAPER-VAULT/<slug>/figures/* <slug>/
   cp ~/PAPER-VAULT/<slug>/动态效果.html <slug>/  # 如果有
   git add <slug>/
   git commit -m "图床: <slug> - <N> 图 + 1 HTML (来自 MiniCPM5-1B / etc)"
   git push
   ```

## 引用规则 (PODCAST 文案)

撰写 wow-podcast 稿子时, **优先用 CDN URL**, 不用本地相对路径。读者打开 .md 看不到图 = 阅读体验断档。

✅ 推荐:
```markdown
![训练全景图](https://cdn.jsdelivr.net/gh/seleman66eeddwegger3-art/wow-podcast-assets@main/2506.07900/training_recipe.png)
```

❌ 反模式:
```markdown
![训练全景图](figures/training_recipe.png)  # 只在本地 vault 有效
```
